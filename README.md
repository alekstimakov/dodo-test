## Что реализовано

- Детекция **людей** класса `person` через `YOLOv8n`
- Анализ только **одного столика**
- Интерактивный выбор ROI по точкам:
  - `LMB` добавить точку
  - `R` откатить последнюю точку
  - `C` очистить выделение
  - `Enter` подтвердить
- Возможность передать полигон через `--roi-json`:
  - JSON-строка
  - JSON-файл
- Три состояния столика:
  - `EMPTY`  человека нет в зоне стола и рядом
  - `APPROACH`  человек находится в `near-zone` рядом со столом если стол был 'Empty'
  - `OCCUPIED`  человек находится внутри ROI стола
- Антидребезг смены состояний через `state_hold_seconds`
- Лог событий в `events.csv`
- Диагностические поля для последующего разбора
- Базовая аналитика:
  - задержка между уходом гостя `OCCUPIED -> EMPTY`
  - и следующим появлением человека `APPROACH` или `OCCUPIED`
- Таймстемпы событий привязаны к кадрам инференса, что дает более точную офлайн-аналитику при `process-every > 1`
- Визуализация в `output.mp4`:
  - контур стола меняет цвет по состоянию
  - человек выделяется bbox
  - на кадре отображается `avg_delay`

## Логика работы

Для детекции используется готовая модель **YOLOv8n** из библиотеки **Ultralytics**.  
Скрипт детектирует только людей и дальше проверяет их положение относительно выбранного столика.

Логика состояний такая:

- `OCCUPIED` фиксируется только когда человек действительно находится в зоне столика
- для этого используется комбинация опорной точки и пересечения `bbox` с ROI
- `APPROACH` определяется по появлению человека в `near-zone` вокруг столика если стол был 'Empty'
- `EMPTY` фиксируется, когда человека нет ни внутри столика, ни рядом с ним

Такой подход позволяет отделять человека, который реально находится у стола, от человека, который просто проходит рядом.

Чтобы уменьшить ложные переключения состояний, используется удержание состояния в течение заданного времени через параметр `state_hold_seconds`.

## Базовая аналитика

Во время обработки все события записываются в `events.csv`.

Для каждого случая, когда стол после состояния `OCCUPIED` становится пустым, скрипт ищет первое следующее событие `APPROACH` или `OCCUPIED` и считает задержку между ними.

Итоговая метрика:

**среднее время между уходом гостя и подходом следующего человека**

После завершения обработки в консоль выводятся:

- количество найденных пар `departure -> next person`
- средняя задержка `Average delay`
