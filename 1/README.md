# Прототип системы детекции уборки столиков по видео

## О проекте

Это упрощенный прототип видеоаналитики для одного выбранного столика на одном видео.
Проект показывает полный базовый пайплайн:

1. выбрать столик вручную;
2. детектировать людей на видео;
3. определить состояние столика (`EMPTY`, `APPROACH`, `OCCUPIED`);
4. зафиксировать события в таблицу;
5. посчитать задержку между уходом и следующим подходом.

## Цель

Сделать рабочий MVP без обучения собственной модели, но с понятной логикой событий и воспроизводимым запуском.

## Что делает скрипт

Скрипт `main.py`:

- обрабатывает одно видео;
- работает с одним выбранным столиком;
- строит `near-zone` вокруг столика автоматически (или берет из `--near-roi-json`);
- детектирует только класс `person` через YOLOv8 (Ultralytics);
- определяет состояние столика по положению человека относительно зон;
- пишет события в `events.csv`;
- сохраняет видео с визуализацией в `output.mp4`.

## Состояния

- `EMPTY` — людей нет ни в зоне столика, ни рядом.
- `APPROACH` — человек появился после периода пустоты (этап подхода).
- `OCCUPIED` — человек находится в зоне столика.

## Логика детекции

1. На каждом N-м кадре запускается YOLO (`--process-every`).
2. Учитываются только детекции `person`.
3. Для каждой детекции считается попадание в зону столика и near-зону.
4. Из признаков `has_inside/has_near` вычисляется сырое состояние.
5. `OccupancyStateMachine` применяет антидребезг (`--state-hold-seconds`), чтобы убрать шумные переключения.
6. При смене стабильного состояния событие пишется в CSV.

## Аналитика

Записываются временные метки всех событий.

Базовая метрика:

**среднее время между уходом гостя и подходом следующего человека**

Расчет:

- старт отсчета: переход `OCCUPIED -> EMPTY`;
- конец отсчета: первое следующее событие `APPROACH` или `OCCUPIED`.

## Визуализация

В `output.mp4`:

- контур столика меняет цвет по состоянию;
- рисуются боксы людей;
- отображается `avg_delay`.

## Структура результата

После запуска формируются:

- `output.mp4` — видео с визуализацией;
- `events.csv` — таблица событий.

## Полный гайд запуска

### 1. Требования

- Python 3.8+
- Windows/Linux/macOS

### 2. Установка зависимостей

Откройте терминал в папке `1` и выполните:

```bash
pip install -r requirements.txt
```

### 3. Подготовка файлов

Убедитесь, что в папке есть:

- входное видео (например, `video1.mp4` или `input.mp4`);
- веса модели (`yolov8n.pt`) или путь к своей модели через `--model`.

### 4. Базовый запуск

```bash
python main.py --video video1.mp4
```

Если `--roi-json` не задан, откроется окно для ручного выбора полигона столика.

Управление в окне выбора ROI:

- `LMB` — добавить точку;
- `R` — удалить последнюю точку;
- `C` — очистить;
- `Enter` — подтвердить.

### 5. Запуск с явными выходными файлами

```bash
python main.py --video video1.mp4 --output output.mp4 --events-csv events.csv
```

### 6. Headless/повторяемый запуск с готовым ROI

```bash
python main.py --video video1.mp4 --roi-json "[[100,120],[300,120],[320,260],[90,250]]"
```

или:

```bash
python main.py --video video1.mp4 --roi-json roi.json
```

Пример `roi.json`:

```json
{
  "points": [[100,120], [300,120], [320,260], [90,250]]
}
```

### 7. Настройка near-зоны

Near-зона по умолчанию строится автоматически вокруг столика:

- `--near-zone-scale` — коэффициент расширения;
- `--near-zone-max-offset-px` — ограничение максимального расширения в пикселях.

Пример:

```bash
python main.py --video video1.mp4 --near-zone-scale 1.10 --near-zone-max-offset-px 40
```

### 8. Проверка результата

- откройте `output.mp4` и проверьте корректность состояний;
- откройте `events.csv` в Excel/Pandas для анализа событий.

## Полезные параметры

- `--process-every` — запуск детектора раз в N кадров;
- `--conf-threshold` — порог confidence;
- `--state-hold-seconds` — антидребезг переключения состояний;
- `--near-zone-scale` — масштаб near-зоны;
- `--near-zone-max-offset-px` — максимальное расширение near-зоны;
- `--device auto|cpu|cuda` — устройство для инференса.

### Тема

**Прототип системы детекции уборки столиков по видео**

### Цель

Создать упрощенный, но рабочий прототип на одном видео, демонстрирующий основной пайплайн обработки.

### Требования

1. Детекция событий (основное):
- использовать детекцию движения (OpenCV) или готовую YOLO-модель (например, YOLOv8n) для людей;

- записывать временные метки событий в Pandas DataFrame;
- для каждого `EMPTY` после `OCCUPIED` считать время до следующего подхода;
- вывести среднюю задержку.

3. Визуализация (минимум):
- отрисовать зону столика на видео;
- менять цвет зоны по состоянию;
- по желанию вывести отчет в консоль/файл.
